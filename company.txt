  - التحديد (sidebar): Dashboard, Generate Data, Datasets, Subscription
  - شركة لا ترى توصيات المحاصيل (هذا للمزارع فقط)

      3. توليد Humidity متعلقة بـ Rainfall (ارتباط موجب)
      4. توليد pH ضمن نطاق المحصول

3. datasets.html - أرشيف مجموعات البيانات (مشترك)
================================================================================

📋 الوظيفة:
  - عرض جميع مجموعات البيانات المُنشأة مع نتائج التحقق
  - فتح مجموعة بيانات للعرض التفصيلي
  - حذف مجموعة بيانات

🔘 الأزرار:
  - View -> يفتح dataset_viewer.html?id=X لعرض تفاصيل المجموعة
  - Delete -> يحذف المجموعة من localStorage ويعيد تحميل الصفحة
  - Pagination -> 10 مجموعات لكل صفحة

⚙️ كيف تعمل تقنياً:
  - تجلب من DataService.getGenerations() (localStorage 'synthai_generations')
  - لكل مجموعة تعرض:
    • Date -> وقت الإنشاء
    • Rows -> عدد الصفوف
    • Level -> Medium / Large / Big Data (مع badge لوني)
    • Quality -> % (أخضر >=85%, برتقالي >=70%, أحمر <70%)
    • Similarity -> % نسبة التشابه مع البيانات الحقيقية
    • Status -> Valid (إذا quality >= 80) أو Review
    • Actions -> View / Delete
  - الباجينيشن: 10 entries/page

4. dataset_viewer.html - عرض مجموعة بيانات (مشترك)
================================================================================

📋 الوظيفة:
  - عرض تفصيلي لمجموعة بيانات محددة
  - جدول تفاعلي مع بحث وتصفية
  - إحصائيات الأعمدة والارتباطات
  - تصدير كـ CSV

🔘 الأزرار:
  - Search -> يصفّي الجدول حسب النص المدخل (كل الأعمدة)
  - Download CSV -> يصدر البيانات كملف CSV (كل الصفوف)
  - Pagination -> 20 صف لكل صفحة

⚙️ كيف تعمل تقنياً:
  - تجلب id من URL parameter (window.location.search)
  - DataService.getGenerationById(id) تجلب البيانات
  - إذا full data موجود في _fullDatasetCache -> تستخدمه للعرض والتصدير
  - إذا لا -> ترجع الـ preview (أول 50 صف)
  - exportCSV() تستخدم fullData إذا موجود، أو preview.data
  - إحصائيات الجانب:
    • Dataset Trust Score: qualityScore
    • Correlation Match: % من computeCorrelationSimilarity()
    • Statistical Similarity: % من computeSimilarityScore()
    • Validation Checks: قائمة الفحوصات
    • Column Statistics: Mean, Std, Min, Median, Max لكل عمود
    • Correlation Heatmap
    • Data Completeness

5. subscription.html - إدارة الاشتراك (مشترك)
================================================================================

📋 الوظيفة:
  - عرض خطط الاشتراك الثلاث
  - مقارنة الميزات بين الخطط
  - الترقية عن طريق محاكاة الدفع

🔘 الأزرار:
  - الزر لكل خطة: Free ($0)، Pro ($9.99/شهر)، Premium ($29.99/شهر)
  - Confirm Payment -> يؤكد الدفع ويرقي الخطة
  - Cancel -> إلغاء

⚙️ كيف تعمل تقنياً:
  - PaymentService.getAvailablePlans() ترجع تعريفات الخطط
  - لكل خطة تعرض: السعر، الميزات، badge
  - الضغط على "Get Started":
    • Free: ترقية فورية
    • Pro/Premium: تظهر modal الدفع
  - simulatePayment():
    • توليد payment_id
    • PaymentService.subscribe() تخصّص user.plan
    • صلاحية 30 يوماً مع expiresAt
  - getUserPlan(userId) ترجع الخطة الحالية

كيف يعمل الموقع من البداية للنهاية؟
================================================================================

1. المستخدم يفتح http://127.0.0.1:8080/pages/login.html
2. يسجل دخول (AuthService.login)
3. التطبيق يعيد التوجيه حسب role (admin_home / farmer_home / company_home)
4. Sidebar يظهر الروابط المناسبة للدور
5. كل صفحة تحمل بياناتها من الـ Services
6. Services تقرأ/تكتب في localStorage (تخزين محلي في المتصفح)
7. إذا Backend شغال (+8001)، API يستخدم CTGAN حقيقي
8. إذا Backend مش شغال، JS simulation يغطي كل الوظائف

التخزين:
  localStorage['synthai_users']       -> المستخدمين
  localStorage['synthai_session']     -> الجلسة الحالية (في sessionStorage فعلياً)
  localStorage['synthai_generations'] -> مجموعات البيانات
  localStorage['synthai_history']     -> سجل التنبؤات
  localStorage['synthai_payments']    -> المدفوعات
  localStorage['synthai_subscriptions'] -> الاشتراكات
  _fullDatasetCache (in-memory)       -> بيانات كاملة (لتفادي limit 5MB)
