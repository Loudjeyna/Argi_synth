======================
المتطلبات الوظيفية (Functional Requirements)
======================

--- متطلبات المزارع (Farmer Requirements) ---

1. نظام المصادقة (Authentication)
   - تسجيل الدخول باستخدام اسم المستخدم وكلمة المرور
   - تسجيل الخروج
   - التحقق من الدور (role-based access)

2. التنبؤ بالمحاصيل (Crop Prediction)
   - عرض نموذج إدخال بيانات التربة والمناخ (N, P, K, temperature, humidity, pH, rainfall)
   - إرسال البيانات إلى نظام التوصية بالذكاء الاصطناعي
   - عرض نتائج التنبؤ بأفضل 3 محاصيل مناسبة مع نسب الثقة
   - عرض توصيات مفصلة لكل محصول مقترح

3. مقارنة التربة والمحاصيل (Soil vs Crop Comparison)
   - عرض جدول مقارنة بين ظروف التربة الحالية والمتطلبات المثلى للمحاصيل
   - إظهار الفروقات بين القيم الحالية والقيم المثلى

4. متطلبات المحاصيل (Crop Requirements)
   - عرض بطاقات معلومات لكل محصول تحتوي على:
     - درجة الحرارة المثلى
     - الرطوبة المثلى
     - مستوى الأس الهيدروجيني (pH)
     - كمية الأمطار المناسبة
     - العناصر الغذائية المطلوبة (N, P, K)
   - تصفية المحاصيل حسب النوع

5. عرض توصيات المحاصيل (Crop Recommendations)
   - البحث عن محاصيل حسب ظروف معينة (conditions_crop)
   - البحث عن الظروف المثلى لمحصول معين (crop_conditions)
   - عرض النتائج بشكل منظم

6. لوحة التحكم الرئيسية (Farmer Dashboard)
   - عرض ملخص سريع عن النظام
   - روابط سريعة للوصول إلى التنبؤ والمقارنة
   - عرض إحصائيات عامة

--- متطلبات الشركة (Company Requirements) ---

1. نظام المصادقة (Authentication)
   - تسجيل الدخول باستخدام اسم المستخدم وكلمة المرور
   - تسجيل الخروج
   - التحقق من الدور (role-based access)

2. معاينة البيانات (Data Viewer / Dataset Viewer)
   - عرض مجموعات البيانات المخزنة في النظام
   - تصفح البيانات مع ترقيم الصفحات (pagination)
   - عرض إحصائيات الأعمدة (min, max, mean, median)
   - عرض مصفوفة الارتباط (correlation matrix)
   - البحث والتصفية داخل مجموعة البيانات
   - عرض مؤشر جودة البيانات (quality ring)

3. تقييم البيانات (Data Evaluation)
   - عرض تقييم لجودة البيانات الأصلية
   - عرض تقييم لجودة البيانات المُنشأة (synthetic)
   - مقارنة بين البيانات الأصلية والبيانات المُنشأة
   - عرض رسوم بيانية ومخططات للمقارنة

4. توليد البيانات (Generate Data)
   - اختيار نوع مجموعة البيانات: محصول (crop)، تربة (soil)، طقس (weather)
   - اختيار حجم البيانات: وسيط (1,000)، كبير (10,000)، ضخم (100,000)
   - توليد بيانات اصطناعية باستخدام SDV CTGAN
   - معاينة البيانات المُنشأة قبل التحميل
   - تحميل البيانات كملف CSV
   - عرض سجل التوليد السابق مع إمكانية التحميل

5. عرض سجل البيانات (History / Datasets)
   - عرض سجل جميع عمليات التوليد السابقة
   - تحميل البيانات المُنشأة سابقاً
   - عرض تفاصيل كل مجموعة بيانات

6. زيادة البيانات (Data Augmentation) - Company
   - رفع ملف CSV موجود
   - زيادة حجم البيانات الموجودة بتوليد بيانات اصطناعية إضافية
   - معاينة النتائج وتحميلها

7. لوحة التحكم الرئيسية (Company Dashboard)
   - عرض ملخص عن البيانات المتاحة
   - روابط سريعة للتوليد والتقييم

--- متطلبات المدير (Admin Requirements) ---

1. نظام المصادقة (Authentication)
   - تسجيل الدخول باستخدام اسم المستخدم وكلمة المرور
   - تسجيل الخروج
   - التحقق من الدور (role-based access)

2. إدارة المستخدمين (User Management)
   - عرض قائمة بجميع المستخدمين المسجلين
   - إضافة مستخدم جديد (إنشاء حساب)
   - تعديل بيانات المستخدم (الاسم، البريد الإلكتروني، الدور)
   - حذف مستخدم
   - البحث عن مستخدمين
   - تصفية المستخدمين حسب الدور

3. إدارة التدريب (Training Management) - admin_training
   - رفع ملف CSV لبيانات التدريب
   - تنظيف البيانات (data cleaning) - إزالة القيم المكررة والمفقودة
   - تكوين معلمات النموذج (model configuration):
     - عدد الحقب (epochs)
     - حجم الدفعة (batch size)
     - بُعد الطبقات المخفية (hidden dimension)
   - إضافة مهمة تدريب جديدة (Add Training)
   - مراقبة حالة التدريب (قيد التشغيل / مكتمل / فاشل)

4. إدارة النماذج (Model Management) - admin_models
   - عرض قائمة بجميع النماذج المدربة
   - البحث عن نماذج
   - تصفية النماذج حسب الحالة أو التاريخ
   - عرض تفاصيل النموذج

5. تفاصيل النموذج وتوليد البيانات (Model Details & Generation) - admin_model_details
   - عرض تفاصيل النموذج المدرب (المعالم، تاريخ التدريب، الحالة)
   - توليد بيانات اصطناعية من نموذج مدرب
   - اختيار عدد الصفوف المراد توليدها
   - معاينة البيانات المُنشأة
   - تحميل البيانات كملف CSV

6. توليد البيانات (Data Generation) - Admin
   - توليد بيانات اصطناعية مباشرة بدون نموذج
   - اختيار نوع البيانات وحجمها
   - معاينة وتحميل النتائج

7. زيادة البيانات (Data Augmentation) - Admin
   - رفع ملف CSV وزيادة حجمه
   - توليد بيانات إضافية مشابهة للبيانات المرفوعة

8. تقييم البيانات (Data Evaluation)
   - تقييم جودة البيانات الأصلية والمُصنّعة
   - عرض مقارنات وإحصائيات

9. عرض سجل البيانات (History)
   - عرض جميع عمليات التوليد السابقة على مستوى النظام
   - تحميل البيانات المُنشأة سابقاً

10. إدارة الاشتراكات (Subscription Management)
    - عرض خطط الاشتراك المتاحة
    - إدارة اشتراكات المستخدمين

11. لوحة التحكم الرئيسية (Admin Dashboard)
    - عرض إحصائيات النظام (عدد المستخدمين، عدد النماذج، عدد مجموعات البيانات)
    - عرض نشاط النظام الأخير
    - روابط سريعة للإدارة


======================
المتطلبات غير الوظيفية (Non-Functional Requirements)
======================

1. الأداء (Performance)
   - يستجيب النظام لطلبات المستخدم خلال 3 ثوانٍ كحد أقصى
   - دعم توليد بيانات ضخمة (100,000 صف) بشكل فعال
   - استخدام التخزين المؤقت (localStorage/sessionStorage) لتحسين سرعة التصفح
   - استخدام تقنية circuit-breaker لإدارة فشل الاتصال بالخادم

2. التوفر (Availability)
   - يعمل النظام بنمطين: Frontend فقط (وضع عدم الاتصال) أو Full Stack (مع Backend)
   - دعم التبديل التلقائي إلى المحاكاة المحلية (JS simulation) عند تعطل API
   - آلية health check للـ API مع مهلة 3 ثوانٍ

3. الأمان (Security)
   - نظام مصادقة مزدوج: تخزين بيانات المستخدمين في localStorage و sessionStorage
   - التحقق من الدور (role-based access control) في كل صفحة
   - حماية الصفحات من الوصول غير المصرح به (إعادة التوجيه إلى صفحة تسجيل الدخول)
   - عدم تخزين كلمات المرور كنص عادي (حيثما أمكن)
   - حماية مسارات API من الوصول غير المصرح به

4. سهولة الاستخدام (Usability)
   - واجهة مستخدم باللغة العربية مع دعم RTL (من اليمين إلى اليسار)
   - دعم التبديل بين اللغات (العربية، الفرنسية، الإنجليزية)
   - تصميم متجاوب (responsive) يدعم جميع أحجام الشاشات
   - قائمة جانبية واضحة مع أيقونات لكل دور
   - رسائل خطأ واضحة ومفهومة باللغة العربية

5. الصيانة والتوسع (Maintainability & Scalability)
   - هيكل معياري (modular) يفصل بين frontend و backend و core logic
   - استخدام نمط Singleton في المكونات المهمة (مثل CropRecommender)
   - دعم إضافة محاصيل جديدة دون تعديل الكود الأساسي
   - دعم إضافة أنواع بيانات جديدة بسهولة

6. الموثوقية (Reliability)
   - معالجة الأخطاء بشكل مناسب في جميع طبقات النظام
   - التحقق من صحة البيانات المدخلة قبل المعالجة
   - ضبط batch_size تلقائياً ليكون متوافقاً مع PAC (قابل للقسمة على 10)
   - وضع fallbacks في حال فشل الخادم

7. قابلية النقل (Portability)
   - يعمل على أنظمة التشغيل: Windows، Linux، macOS
   - لا يتطلب سوى Python 3.8+ ومكتبات مفتوحة المصدر
   - دعم التشغيل المباشر دون خادم (Frontend only mode عبر HTTPServer)

8. الخصوصية (Privacy)
   - تخزين بيانات الجلسة محلياً في المتصفح (sessionStorage)
   - مسح بيانات الجلسة تلقائياً عند تسجيل الخروج
   - عدم مشاركة بيانات المستخدم مع أطراف ثالثة

9. التوافقية (Compatibility)
   - دعم المتصفحات الحديثة (Chrome, Firefox, Edge, Safari)
   - دعم REST API للتواصل مع الخادم
   - دعم تنسيق CSV لتصدير واستيراد البيانات
